# Archive Rules

Status: canonical  
Owner: program-control  
Last updated: 2026-03-04  
Canonical source: `docs/program/doc_governance.md`

`docs/archive/` stores historical records only.

## Rules

- Archive run logs, gap reports, superseded plans, one-off migration reports, and closed execution snapshots here.
- Do not treat anything in `docs/archive/` as live planning state.
- Keep the live planning surface in `docs/program/`.
- Prefix archived snapshots with the original filename and a date when you create new historical records.
- Leave historical cleanup and deprecation records in archive scope only. Do not link them from canonical planning entrypoints except from a dedicated archive index.
